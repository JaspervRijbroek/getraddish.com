This is most likely the easiest way to override some of the variables in the system.
At the moment this only works on controllers and tables.

When creating a component and a table/ controller only needs some behaviors you can create a config.json in the root of the component.
This file will automatically be used. One might look like this:

{% highlight javascript linenos %} { "controllers": { "foo": { "behaviors": { "uploadable": {} } } }, "tables": { "foo": { "behaviors": { "socketable": {}, "home:foo.database.table.behavior.bar": {} }, "name": "foo_bar_baz" } } }{% endhighlight %}

As you can see we bound behaviors to the foo controller,
and added behaviors to the foo table and set the table name it has to use to foo_bar_baz without having to create a table override.

The added behaviors are key, value so the key is the behavior to load and the value is the config for the behavior.

This might come in handy if this is the only thing you want to override.
When needed more specific overrides you need to override the object.
