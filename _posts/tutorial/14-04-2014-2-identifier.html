Identifiers

Identifiers are a huge advantage in this framework.
With them you are able to load all kinds of objects, from controllers, to views and behaviours wherever and whenever.

As an example I want to receive the model items of the component menu in the application foo I will use this:
Service.get('foo:menu.model.item') This will return the promise of the items model.
This can also be done for the controller eg: Service.get('foo:menu.controller.item')

Lets say there is something you have created in another application, you can reach this as well, just change the first parameter of the identifier
eg: Service.get('bar:menu.model.item')

A simple way to define the identifiers is: Service.get('<application>:<component>.<path>.<name>') here path is a dot separated path.
Another example but then for the table row Service.get('bar:menu.database.row.item')
If the object isn't present don't worry it will return a fallback.

PS: When outside of an object you can use Service.get when inside an object you can use this.getService
