The dispatcher is the handler of request and response and thus will handle the output accordingly.
Also when there is no view given in the request the dispatcher will assume a view which is by default the plural of the component name

When you want this can be overridden by adding the config variable controller in the initialize function of the dispatcher, below an example. {% highlight javascript linenos %} var Dispatcher = require('raddish').Dispatcher; var util = require('util'); function ExampleDispatcher(config) { ExampleDispatcher.super_.call(this, config); }; util.inherits(ExampleDispatcher, Dispatcher); ExampleDispatcher.prototype.initialize = function(config) { config.controller = 'foo'; return ExampleDispatcher.super_.prototype.initialize.call(this, config); }; module.exports = ExampleDispatcher;{% endhighlight %} In this example we have set the default controller when there is no view found. So when we go to http://example.com/web/example it will automatically show the contents of foo.
And when you go to http://example.com/web/example/bars it will show the content of bars.
