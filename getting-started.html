Getting Started A quick intro to Raddish

In this section we will guide you throught the steps of creating your own application and the basic setup.
For further documentation about the objects and accepted parameters/ return values you can check the API section.

For this introduction we will use the router which is part of the system, which ia very basic,
Check for static files if it is available return it, else try to receive data from the framework.

Urls and the Public folder

By default the system will check the requested application in the default component for a dispatcher
This dispatcher has to be present or else there will be a 404 error.

How do requests look like?
http://example.com/<application>/<component>/<view>
The view is optional when no view is given it will look for the plural of the component eg: component name = menu view name = menus.

You can also send get variables as states into the system. The states given to the system will also be given back in the response.
Also the will make a difference between singular and plural views, plurals will return a list and singulars will return or an empty object if it doen't exists or a filled row object.

So what is the deal with the public folder?
This is the folder where all the public files are located. You can access them as expected eg: http://example.com/css/foo.css
will try to get file from the public folder, if it can't be found the request will be handled as a framework request.

Setting up the system

First we have to create a two files the application file app.js and config.js the file config.js will hold all the config options
like default output format and databases.

The app.js is very simple below is an example to use in a project. {% highlight javascript linenos %} var Router = require('raddish').Router; var http = require('http'); http.createServer(function(req, res) { new Router().route(req, res); }).listen(1337); console.log('Server running on port: 1337');{% endhighlight %}

This will create a http server which will listen to requests, when you request a file or data from the application the router will handle the request.
When the file is found it will return it.

Next is the config.js where all the configuration options are set. Below I will give an example to use. {% highlight javascript linenos %} module.exports = { "db": { "default": { "host": "localhost", "port": 3306, "user": "root", "password": "root", "database": "foo" } }, "layout": "json", "gzip": false }{% endhighlight %}

We will explain the values

db
This object holds all the database connections there must be a default connection
this one will be used by default for other connections you can add ass many as you need/ want.
layout
This is the default format which will be outputted by the system, by default this is JSON
this can be changed to every layout you want. Also if you want a single request to have a different layout
yo can send the get-variable layout in the request with the needed layout type, by default only JSON is in the core.
gzip
Allow the response data to be gzipped to save bandwith data, this might be useful for mobile applications.

This was it for the basic setup and some information about the basics of the system.

Your first application and component

Now we will try to create a simple blogging component called blog.
We won't be handling the frontend because this can be done the way you like it.
But first we have to create an application. The application will be called web. So in our applications folder we create the folder named web.
And in this folder we create the folder components. Unfortunately we still rely on a default dispatcher so we have to create this one first.
In the components folder we create the folder default, this will hold the component wide overrides and can be used for almost everything.
Here we will create the file called dispatcher.js with the following code in there. This is the minimum needed for a dispatcher. 
var Dispatcher  = require('raddish').Dispatcher;
var util        = require('util');

function DefaultDispatcher() {
    DefaultDispatcher.super_.apply(this, arguments);
}

util.inherits(DefaultDispatcher, Dispatcher);

module.exports = DefaultDispatcher;
After this we are almost done, we still have to add the table the tables will hold a naming convention in this case blog_blogs
In this table we have the following columns id, title, content the id column will be a primary key.
After this if you send a POST request to http://example.com/web/blog/blog it will add a table entry
If you send a POST request to http://example.com/web/blog/blog?id=1 it will update the table row with ID 1 with the posted data.

This holds the base of a component, when you don't need any extra overrides the default system will take care of everything.
